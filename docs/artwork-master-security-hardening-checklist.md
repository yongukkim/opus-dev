# OPUS 작품 원본 보안 하드닝 체크리스트

목표: 웹에는 저해상 파생만 노출하고, 원본(고해상)은 앱 인증 경로에서만 접근되도록 강제한다.

## 1) 저장소 경계 분리

- [ ] 원본 경로를 `private/master/*`로 고정하고 웹 공개 경로와 완전 분리
- [ ] 웹 파생 경로를 `public/derivatives/*`로 고정
- [ ] 업로드 시 원본/파생 메타를 별도 필드로 기록(혼용 금지)
- [ ] 운영 스크립트에서 `public/*`에 원본 확장자/해상도 파일 존재 여부 검사

## 2) API 접근 통제

- [ ] 웹 API는 원본 바이트를 반환하지 않도록 정책화(403/404)
- [ ] 앱 전용 API는 소유권 + 기기 바인딩 + 단기 lease 토큰 필수
- [ ] lease 토큰은 짧은 TTL 및 1회성/재사용 제한 정책 적용
- [ ] 원본 다운로드 응답에 `Cache-Control: private, no-store` 적용
- [ ] CDN/프록시에서 원본 경로 캐시 금지

## 3) 인증/권한/소유권 검증

- [ ] Chronicle 기준 현재 소유자 검증 로직을 앱 원본 API에 강제
- [ ] 운영자/아티스트/소장자 권한별 허용 엔드포인트 매트릭스 문서화
- [ ] 만료/탈취 토큰 대응(즉시 revoke, 기기 해제) 절차 준비
- [ ] 관리자 우회 접근은 감사로그 필수 + 사유 기록

## 4) 암호화/시크릿 관리

- [ ] 전송구간 TLS 1.3 강제(외부 노출 구간)
- [ ] 저장구간 암호화(KMS 기반 키 관리) 적용
- [ ] 시크릿은 코드/이미지에 포함하지 않고 Secret Manager로만 주입
- [ ] 원본 접근 서명키/토큰키 로테이션 절차 문서화

## 5) 감사로그/탐지

- [ ] 원본 API 호출 로그에 userId/deviceId/artworkId/requestId 기록
- [ ] 로그에 파일 경로/PII 원문 직접 노출 금지(마스킹)
- [ ] 비정상 패턴(대량 요청, 실패율 급증, 다중 기기 이동) 알림 설정
- [ ] 침해 대응 런북(차단, 토큰 폐기, 키 교체, 포렌식) 점검

## 6) 검증 자동화

- [ ] 배포 후 스모크: 웹 경로에서 원본 요청 시 403/404 확인
- [ ] 배포 후 스모크: 앱 인증 경로만 원본 접근 성공 확인
- [ ] 주간 점검: 스토리지 샘플링으로 원본/파생 경계 위반 탐지
- [ ] 백업 복구 테스트 시 원본 접근 제어 정책 유지 여부 확인
